# Penjelasan Activity Diagram

Pada Gambar 4.3 ditunjukkan *activity diagram* untuk proses login. Aktivitas dimulai ketika pengguna membuka halaman login, kemudian sistem menampilkan form login yang berisi kolom email dan password. Setelah pengguna memasukkan kredensial dan mengirimkan form, sistem melakukan validasi terhadap data yang dimasukkan. Apabila kredensial tidak valid, sistem menampilkan pesan error dan pengguna diarahkan kembali ke form login untuk mengisi ulang data. Apabila kredensial valid, sistem mengidentifikasi role pengguna (Admin, Guru, atau Kepala Sekolah) lalu mengarahkan pengguna ke halaman dashboard yang sesuai dengan role-nya.

Pada Gambar 4.4 ditunjukkan *activity diagram* untuk proses kelola data guru. Aktivitas dimulai ketika Admin membuka menu data guru, kemudian sistem menampilkan daftar guru yang tersimpan di database. Selanjutnya Admin memilih aksi yang akan dilakukan, yaitu tambah, edit, atau hapus data guru. Jika Admin memilih tambah atau edit, sistem menampilkan form isian data guru, kemudian Admin mengisi atau mengubah data yang diperlukan dan menekan tombol simpan. Sistem kemudian memvalidasi data yang dimasukkan; apabila data tidak valid maka sistem menampilkan pesan error, sedangkan apabila data valid maka sistem menyimpan data ke database dan menampilkan notifikasi berhasil. Jika Admin memilih hapus, sistem menampilkan konfirmasi penghapusan; setelah Admin mengonfirmasi, data guru dihapus dari database dan sistem menampilkan notifikasi berhasil.

Pada Gambar 4.7 ditunjukkan *activity diagram* untuk proses kelola kriteria SAW. Aktivitas dimulai ketika Admin membuka menu kriteria penilaian, kemudian sistem menampilkan daftar kriteria beserta bobot yang sudah tersimpan. Admin kemudian memilih kriteria yang akan diubah, lalu mengedit nama, jenis (benefit atau cost), dan bobot kriteria tersebut. Setelah Admin menekan tombol simpan, sistem memvalidasi data yang dimasukkan termasuk memastikan bahwa total bobot seluruh kriteria bernilai 1. Apabila validasi gagal, sistem menampilkan pesan error dan Admin diminta untuk memperbaiki data. Apabila validasi berhasil, sistem menyimpan perubahan kriteria ke database dan menampilkan notifikasi berhasil.

Pada Gambar 4.8 ditunjukkan *activity diagram* untuk proses generate QR Code absensi. Aktivitas dimulai ketika Admin membuka menu generate QR Code, kemudian sistem menampilkan halaman generate dengan daftar guru. Admin memilih guru yang akan dibuatkan QR Code, lalu menekan tombol generate. Sistem kemudian mengenkripsi data absensi menggunakan algoritma AES-256-CBC, men-generate gambar QR Code berdasarkan data yang telah dienkripsi, dan menetapkan masa berlaku QR Code selama 600 detik. Setelah proses selesai, sistem menampilkan QR Code pada layar monitor dalam mode kiosk agar dapat dipindai oleh guru.

Pada Gambar 4.9 ditunjukkan *activity diagram* untuk proses scan QR Code absensi. Aktivitas dimulai ketika Guru membuka menu scan QR Code pada smartphone dan mengarahkan kamera ke QR Code yang ditampilkan pada monitor. Sistem kemudian mendekripsi data QR Code; apabila QR Code tidak valid atau sudah kadaluarsa, sistem menampilkan pesan error dan proses berakhir. Apabila QR Code valid, sistem melanjutkan dengan memvalidasi geolokasi guru menggunakan formula Haversine; jika lokasi guru berada di luar radius sekolah, sistem menampilkan pesan error dan proses berakhir. Jika lokasi valid, sistem memeriksa apakah guru sudah melakukan check-in pada hari tersebut. Apabila belum, sistem mencatat check-in dan menentukan status kehadiran (tepat waktu atau terlambat). Apabila sudah check-in, sistem mencatat check-out. Selanjutnya sistem menyimpan data absensi ke database dan menampilkan notifikasi keberhasilan kepada guru.

Pada Gambar 4.10 ditunjukkan *activity diagram* untuk proses lihat rekap absensi. Aktivitas dimulai ketika pengguna membuka menu rekap absensi, kemudian sistem menampilkan halaman rekap beserta filter pencarian. Pengguna kemudian memilih filter yang diinginkan seperti periode waktu, nama guru, atau status kehadiran, lalu menekan tombol tampilkan. Sistem mengambil data absensi dari database sesuai dengan filter yang dipilih dan menampilkan hasilnya dalam bentuk tabel rekap absensi yang mencakup informasi tanggal, nama guru, waktu check-in, waktu check-out, dan status kehadiran.

Pada Gambar 4.11 ditunjukkan *activity diagram* untuk proses input nilai siswa. Aktivitas dimulai ketika Guru membuka menu input nilai, kemudian sistem menampilkan halaman input nilai. Guru memilih kelas dan mata pelajaran yang akan dinilai, lalu sistem menampilkan daftar siswa dalam bentuk tabel form penilaian. Guru mengisi nilai untuk setiap siswa meliputi nilai tugas, UTS, dan UAS, kemudian menekan tombol simpan. Sistem memvalidasi data nilai yang dimasukkan; apabila data tidak valid maka sistem menampilkan pesan error. Apabila data valid, sistem menghitung nilai akhir berdasarkan bobot yang telah ditentukan, menyimpan seluruh data nilai ke database, dan menampilkan notifikasi berhasil.

Pada Gambar 4.12 ditunjukkan *activity diagram* untuk proses lihat ranking siswa menggunakan metode SAW. Aktivitas dimulai ketika pengguna membuka menu ranking siswa, kemudian sistem menampilkan halaman ranking beserta filter. Pengguna memilih kelas dan tahun ajaran yang diinginkan, lalu menekan tombol tampilkan ranking. Sistem kemudian menjalankan proses perhitungan SAW yang terdiri dari beberapa tahap, yaitu membangun matriks keputusan berdasarkan kriteria C1 sampai C4, melakukan normalisasi matriks, dan menghitung nilai preferensi (skor SAW) untuk setiap siswa. Setelah perhitungan selesai, sistem mengurutkan siswa berdasarkan skor tertinggi dan menampilkan tabel ranking beserta detail perhitungan SAW.

Pada Gambar 4.14 ditunjukkan *activity diagram* untuk proses export laporan. Aktivitas dimulai ketika pengguna membuka menu laporan, kemudian sistem menampilkan halaman laporan beserta filter. Pengguna memilih jenis laporan dan mengatur filter yang diinginkan seperti periode, kelas, dan status, kemudian memilih format export yang diinginkan yaitu Excel atau PDF, lalu menekan tombol export. Sistem kemudian memeriksa format yang dipilih; apabila pengguna memilih format Excel, sistem men-generate file dalam format .xlsx, sedangkan apabila pengguna memilih format PDF, sistem men-generate file dalam format PDF. Setelah file berhasil dibuat, sistem mengunduh file tersebut ke perangkat pengguna.

Pada Gambar 4.15 ditunjukkan *activity diagram* untuk proses lihat dashboard. Aktivitas dimulai ketika pengguna membuka halaman dashboard, kemudian sistem mengidentifikasi role pengguna yang sedang login. Selanjutnya sistem mengambil data statistik dari database yang meliputi data absensi, nilai, siswa, dan guru. Sistem menghitung ringkasan data seperti total guru hadir, rata-rata kehadiran, dan jumlah siswa, kemudian menampilkan dashboard sesuai dengan role pengguna yang berisi grafik, statistik, dan tabel. Pengguna kemudian dapat melihat seluruh informasi yang ditampilkan pada dashboard tersebut.
